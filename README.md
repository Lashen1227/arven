# arven

> From Welsh **arven** — *"to oversee / to keep watch"*
> arven watches over your env vars so you never forget to document them.

**arven** is a CLI that installs a silent pre-commit hook into your project.
Every time you commit, arven checks staged files for new environment variables —
and blocks the commit if any are missing from `.env.example`.

It also scans your full codebase to generate and maintain `.env.example` automatically.

---

## The Problem

You add `process.env.STRIPE_SECRET_KEY` in a late-night coding session.
You commit. You push. A teammate clones and spends 40 minutes debugging why the app crashes.

The `.env.example` was never updated.

**arven blocks that commit before it ever happens.**

---

## Install

```bash
npm install -g arven
```

---

## Quick Start

Run once per project:

```bash
arven init
```

That's it. arven:
1. Installs a `pre-commit` git hook
2. Scans your codebase and generates `.env.example`
3. From now on — guards every commit silently

---

## How the guard works

When you run `git commit`, arven intercepts it:

```
git commit -m "feat: add stripe payments"
```

If `STRIPE_SECRET_KEY` is new in your staged files but missing from `.env.example`:

```
  arven ✗  Commit blocked

  1 new env var found in staged files but not in .env.example:

    ✗ STRIPE_SECRET_KEY                  ← src/payments/stripe.ts

  Fix with one of:
    arven generate --overwrite   # auto-add to .env.example
    arven diff                   # see full picture

  Then re-commit.
```

If everything is documented — arven stays silent. No noise on clean commits.

## Commands

### `arven init`
Install the hook and generate initial `.env.example` in one step.

```bash
arven init
```

```
  arven — setting up your watcher...

  Step 1/2  Installing pre-commit hook
  ✓ arven pre-commit hook installed.
    → /your/project/.git/hooks/pre-commit

  Step 2/2  Scanning codebase for env vars
  ✓ Generated .env.example (9 variables)

  arven is watching. Every commit is now guarded.
```

---

### `arven generate`
Scan your full codebase and write `.env.example`.

```bash
arven generate
arven generate --overwrite    # replace existing
arven generate -o .env.sample # custom output path
```

```
  arven generate — scanning your codebase...

  ✓ Generated .env.example

  Server
    + BASE_URL                           ← src/server.ts
    + PORT                               ← src/server.ts

  Database
    + DATABASE_URL                       ← src/db/client.ts
    + REDIS_URL                          ← src/db/cache.ts

  Auth & Security
    + JWT_SECRET                         ← src/middleware/auth.ts
    + SESSION_SECRET                     ← src/middleware/auth.ts

  AI / LLM
    + ANTHROPIC_API_KEY                  ← src/services/ai.ts
    + OPENAI_API_KEY                     ← src/services/ai.ts

  Payments
    + STRIPE_SECRET_KEY                  ← src/payments/stripe.ts

  9 variables across 4 groups.
```

Generated `.env.example`:

```env
# Generated by arven — https://github.com/yourusername/arven
# arven (Welsh: "to oversee / to keep watch") watches over your env vars.
# Edit values below before running the project.

# Auth & Security
JWT_SECRET=
SESSION_SECRET=

# Database
DATABASE_URL=
REDIS_URL=

# AI / LLM
ANTHROPIC_API_KEY=
OPENAI_API_KEY=

# Payments
STRIPE_SECRET_KEY=

# Server
BASE_URL=
PORT=
```

---

### `arven check`
CI mode — exits `1` if `.env.example` is out of sync.

```bash
arven check
```

```
  arven check — verifying .env.example sync...

  ✗ 1 variable missing from .env.example:

    - STRIPE_SECRET_KEY

  Run: arven generate --overwrite to fix.
```

Add to your GitHub Actions pipeline:

```yaml
- name: Check .env.example is up to date
  run: npx arven check
```

---

### `arven diff`
See exactly what's missing or stale — without writing any files.

```bash
arven diff
```

```
  arven diff — comparing codebase vs .env.example...

  Missing in .env.example (in code, not documented):

    + STRIPE_SECRET_KEY                  ← src/payments/stripe.ts

  Unused in .env.example (documented, not in code):

    - OLD_LEGACY_API_KEY

  Summary: 8 synced  1 missing  1 unused
```

---

### `arven uninstall`
Remove arven from the git hook.

```bash
arven uninstall
```

## What arven detects

| Syntax | Framework |
|--------|-----------|
| `process.env.VAR` | Node.js / Express / NestJS |
| `process.env['VAR']` | Node.js |
| `import.meta.env.VAR` | Vite / SvelteKit / Next.js |
| `Deno.env.get('VAR')` | Deno |
| `Bun.env.VAR` | Bun |

---

## How arven groups variables

| Group | Matches |
|-------|---------|
| Database | `DB_`, `DATABASE_`, `MONGO`, `POSTGRES`, `REDIS` … |
| Auth & Security | `JWT_`, `AUTH_`, `SECRET`, `PASSWORD` … |
| Cloud & Storage | `AWS_`, `S3_`, `AZURE_`, `CLOUDFLARE_` … |
| Email | `SMTP_`, `MAIL_`, `SENDGRID_`, `RESEND_` … |
| Payments | `STRIPE_`, `PAYPAL_`, `LEMON_` … |
| AI / LLM | `OPENAI_`, `ANTHROPIC_`, `GEMINI_` … |
| Frontend / Public | `NEXT_PUBLIC_`, `VITE_`, `REACT_APP_` … |
| Server | `PORT`, `HOST`, `BASE_URL` … |
| OAuth | `GOOGLE_`, `GITHUB_`, `DISCORD_` … |
| Monitoring & Logging | `SENTRY_`, `DATADOG_`, `LOGTAIL_` … |
| General | Everything else |

## Works with all git interfaces

arven hooks into git itself — not your editor.

| Tool | Works? |
|------|--------|
| Terminal (`git commit`) | ✅ |
| VS Code Source Control | ✅ |
| IntelliJ / WebStorm | ✅ |
| GitHub Desktop | ✅ |
| Fork / Tower | ✅ |

> **Note:** `git commit --no-verify` bypasses all hooks.
> For full protection, combine with `arven check` in CI.
